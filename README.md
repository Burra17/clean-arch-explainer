# Clean Architecture & CQRS Explainer

An interactive, animated step-by-step visualization that shows how a GET request flows through a .NET application built with Clean Architecture and the CQRS pattern (MediatR).

## Overview

The explainer walks through 9 steps — from an initial HTTP request all the way down to the database and back — highlighting which architectural layer is active at each stage. Each step includes a code snippet and a plain-language explanation of what's happening and why.

**Layers covered:**

- **Presentation** — ASP.NET Controller receives the request and delegates via MediatR
- **Application** — Query handler orchestrates the use case through repository abstractions
- **Domain** — Entity definitions (Product)
- **Infrastructure** — Repository implementation using Entity Framework Core
- **Database** — SQL generated by EF Core

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) 18+

### Installation

```bash
npm install
npm run dev
```

The app will be available at `http://localhost:5173`.

## Deployment

The repository includes a GitHub Actions workflow (`.github/workflows/deploy.yml`) that automatically builds and deploys to GitHub Pages on every push to `main`.

To enable it:

1. Go to **Settings > Pages** in your GitHub repository
2. Set **Source** to **GitHub Actions**

If you fork this repo, update the `base` path in `vite.config.js` to match your repository name:

```js
base: '/your-repo-name/',
```
